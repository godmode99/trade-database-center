## ระยะที่ 2: ทำ ingestion pipeline แหล่งหลัก
1. FRED (daily → weekly → monthly)
2. MT5 (เริ่ม daily ก่อน, แล้วเพิ่ม timeframe อื่น)
3. Calendar (weekly snapshot + update)
4. CME FedWatch (probabilities ก่อน quotes)

แนวทางทุก pipeline:
- มี retry + timeout + logging
- มี manifest บอกสถานะ run
- ออกแบบให้รันซ้ำแล้วไม่เสียข้อมูล (idempotent)

## ระยะที่ 3: คุณภาพข้อมูลและการมอนิเตอร์
1. Data validation
   - schema check
   - duplicate check
   - freshness check
2. Alerting
   - แจ้งเตือนเมื่อ fetch ล้มเหลว
   - แจ้งเมื่อข้อมูลผิดปกติ
3. Dashboard สรุปสถานะ ingestion

## ระยะที่ 4: การแสดงผลบนเว็บ

2. หน้า time-series และตารางล่าสุด
3. API ภายในสำหรับ query ตาม source/timeframe
